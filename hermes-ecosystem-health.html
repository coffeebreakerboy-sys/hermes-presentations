Three Memory Layers

The studio remembers through three parallel systems — each with a distinct retention model and access pattern.

layer 1 · inbuilt
MEMORY.md + USER.md
Durable facts — founder's creed, company policy, host info, env details. Frozen snapshot per session. Always injected into context. 1,903 chars / 2,200 cap.
layer 2 · hindsight
episodic memory
Semantic search + recall across past sessions. Needs numpy<2 — re-pin after every hermes update. Used for "what did we do about X" cross-session recall.
layer 3 · gbrain
synthesis + graph
30 pages, 38 chunks, hybrid search + keyword + multi-query. MCP tools: think, query, traverse graph, extract facts. Dream cycle at 4am daily auto-consolidates. 15-min maintainer cron.
# Memory flow: query → gbrain (synthesis) → hindsight (episodic context) → inbuilt (durable policy) $ hindsight_recall "what did we decide about X" # cross-session recall $ gbrain think question="multi-hop synthesis" # graph-aware reasoning $ gbrain put_page slug="..." # write durable page + embed
